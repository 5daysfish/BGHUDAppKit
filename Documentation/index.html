Compiling

OS X Snow Leopard 10.6

1
Click Build, your finished.

OS X Leopard 10.5

1
Expand the "Targets" group item.

2
Double-Click on BGHUDAppKitPlugin to open the Target BGHUDAppKitPlugin Info window.

3
Select the Build tab.

4
Select All Configurations from the Configuration drop down.

5
Change the Architectures setting to 32-Bit Universal

The reason for this is that Interface Builder on OS X Leopard 10.5 doesn't have 64-Bit available.

OS X Tiger and below

BGHUDAppKit does not work on any OS X below 10.5 due to the unavailability of NSGradiant.

Usage

Including in Project

1
Right-Click on your Frameworks group in XCode.

2
Select Add -> Existing Frameworks...

3
Navigate to your BGHUDAppKit build/Release directory and select BGHUDAppKit.framework (Yes, it's a folder)

4
Right-Click on your projects Target (This is generally a target with the same name as your project) located in your Targets group in XCode

5
Select Add -> New Build Phase -> New Copy Files Build Phase

6
Double-Click the new item named Copy Files under your projects target grouping. (You may need to expand your projects target to see this)

7
Select Frameworks from the Destination drop down, close the window when your done.

8
Drag BGHUDAppKit.framework from Frameworks group in XCode to the Copy Files build phase under Targets group in XCode.

9
Build your project.

You should now be able to inspect your products .APP bundle and see a new folder named Frameworks that contains BGHUDAppKit.
